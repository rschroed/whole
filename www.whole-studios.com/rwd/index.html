What is it?

It's designing & building a website so that it responds to the environment in which it's being displayed.

Not one design for all devices or designing for each device.

Usually this means using the same HTML regardless of the device (desktop, tablets, phones) and asking the browser to adjust the presentation (via CSS) based on width of the viewport.

Viewport? That's the visible portion of a web browser.

So, Why RWD?

iPhones are quite popular these days

  • Mobile + Tablet usage > Desktop soon
  • Phones are the primary web browser for 17% of people
  • Links don't open apps
  • Google recommends a single site

but...

The devices themselves are so good at scaling websites, why is this necessary?

One design for
all devices

Sending the same presentation to all devices ignores context

  • You end up with a lot of zooming in and out
  • UI Elements won't be optimized for chubby fingers
  • Users *could* want different things depending on context

Remember fluid design?

Viewports are so different these days that fluid design is hard — or at least ugly.

Designing for each device

Requires some type of device detection.

Also, don't do it.

So, Why RWD?

RWD allows you to be fluid at certain width, fixed at others and makes it easy to show or hide certain elements depending on context.

Mediaqueries


/* Large desktop */
@media (min-width: 1200px) { ... }
 
/* Portrait tablet to landscape and desktop */
@media (min-width: 768px) and (max-width: 979px) { ... }
 
/* Landscape phone to portrait tablet */
@media (max-width: 767px) { ... }
 
/* Landscape phones and down */
@media (max-width: 480px) { ... }
